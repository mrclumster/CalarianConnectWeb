01 — Core Contributions

What I Built

Face Recognition Kiosk

Built the AI system that lets citizens walk up to the barangay entrance and be identified in 2–3 seconds — no ID card needed. It also detects and rejects attempts to use a printed photo or screen to cheat the system (what developers call anti-spoofing), built using Python and the InsightFace library.

Python · InsightFace Anti-Spoofing Intel RealSense

Data Security & Privacy

Set up database-level access controls (called Row-Level Security) so staff and citizens can only ever see data that belongs to them. Caught and fixed a security gap where one citizen could access another's uploaded government ID — and ensured the whole system is compliant with RA 10173, the Philippine Data Privacy Act.

Row-Level Security (RLS) RA 10173 Compliant

Automation & Notifications

Built the backend services (called Edge Functions) that run automatically in the background — sending citizens a phone notification the moment their request is approved, generating official PDF certificates on demand, and powering the voice-to-form feature so residents can simply speak their request using AI (OpenAI Whisper + Claude).

Edge Functions Push Notifications OpenAI · Claude AI

Audit Trail & Accountability

Set up automatic logging using database triggers — every time a citizen's profile is edited or sensitive biometric data is removed, a record is saved: what changed, who made the change, and when. This gives the barangay a clear, tamper-proof history, as required by RA 10173.

Database Triggers Audit Logs RA 10173

Unique Request Tracking Numbers

Discovered and fixed a race condition bug — when two citizens submitted requests at the exact same moment, the system could give them both the same tracking number (e.g. BC-2026-00001). Solved it by moving the numbering to the database using an atomic sequence, so duplicates are impossible even under heavy load.

Race Condition Fix Atomic Sequence

Staff Dashboard & Reports

Wrote the database functions (called RPCs — Remote Procedure Calls) that power the staff dashboard in real time — showing pending requests, approvals, and citizen walk-in counts. Also extended them so super admins can view statistics across all barangays, not just their own.

PostgreSQL RPCs Real-time Stats

03 — Reflection & Growth

What I Learned

"A bug on the visible part of an app breaks a button. A bug in the backend system can expose a citizen's private information." This realization shaped how seriously I approached every task during this internship.

01

The Weight of Data Privacy

Working with real citizens' government IDs and face data made me realize that responsibility goes beyond just completing a task. I started going beyond what was assigned — adding extra safeguards to make sure sensitive information could never be accessed by the wrong person.

02

From Ticket Resolver to Systems Thinker

I started the internship focused on completing the tasks given to me. Over time, I began reviewing the system on my own, spotting gaps that weren't part of any assignment, and putting together a plan to address them — taking ownership rather than just waiting for direction.

03

Applying Knowledge in the Real World

A lot of what I studied in school became meaningful only when I applied it here. Knowing something in theory is different from implementing it on a live system where real people are affected. This internship pushed me to deeply understand the tools I was using, not just use them.

04

Documentation as Engineering Leverage

Before ending the internship, I wrote a detailed handover guide and a deployment manual for the barangay staff. I learned that good documentation is just as important as the work itself — it ensures everything you built continues to be useful long after you're gone.
