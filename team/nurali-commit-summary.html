01 — Core Contributions

What I Built

The kiosk platform that lets citizens walk up to the barangay entrance and be identified in seconds — no ID card needed — then guides them through requesting services entirely on their own.

Face Recognition Pipeline
Mar 10 – 23, 2026

Built the core Python FastAPI server and React kiosk frontend from scratch. Set up the ArcFace recognition pipeline with face-api.js ML models bundled locally, added position validation, stability tracking, and bilingual (TL/EN) guidance for citizens. Configured Vite for Electron packaging.

Python · FastAPI ArcFace face-api.js Electron
Realtime Session Tracking
Mar 11 – 12, 2026

Added Supabase Realtime WebSocket to track citizen sessions and push live updates to the moderator's web panel the instant a citizen is identified. Deployed the send-notification Edge Function and added the barangay_kiosk_settings database migration for per-barangay kiosk configuration.

Supabase Realtime WebSocket Edge Functions
Complete Kiosk Application
Mar 16 – 18, 2026

Completed all kiosk pages — Enrollment, Complaint, FormFill, QueueNumber, and StatusCheck. Added AdminPinModal for secure staff access. Wired the React frontend to the Python face API and created CameraCapture.tsx, api.ts, and realtime.ts so the moderator panel is notified instantly on citizen identification.

React · TypeScript CameraCapture AdminPinModal
Audio Feedback System
Mar 16, 2026

Created sound.ts — citizens now hear audio feedback when identified, on errors, and when a queue number is issued. Refined the Python server and startup script alongside the audio integration to ensure seamless kiosk boot behavior.

05 — Reflection & Growth

What I Learned

Beyond the technical deliverables, this internship reshaped how I think about building software that real people depend on.

"I started thinking about face recognition as just a cool AI problem. By the time I shipped the security fixes, I realized I was holding the front door of a government building in my hands. Every edge case I ignored could mean a citizen being locked out or the wrong person getting in."

Before the Internship

AI hobbyist working on isolated scripts

No production system or real users

Security was an afterthought

No experience with Electron or Supabase

Python and React as separate worlds

→
After the Internship

Shipped a full-stack kiosk used by real citizens

Security-first mindset — resolved 12 findings proactively

Proficient with Electron IPC and Supabase Realtime

Python FastAPI ↔ React frontend integration end-to-end

Confident iterating on live UI from user feedback

🤖

AI in Production is Different

A model that works on test images is not the same as one that works under bad lighting, on every face, every time — with real people waiting.

🔒

Security Can't Be Retrofitted

Patching 12 security issues after the fact took more effort than building them in from the start. I won't make that trade again.

🔗

Full-Stack Ownership

Owning the Python server, the Electron shell, and the React UI simultaneously forced a systems view — no more blaming "the other layer."

🎨

UX for Public Spaces

Designing for a citizen in a busy barangay hall is nothing like designing for a developer. Bilingual text, audio cues, and clear visual guidance matter enormously.
